# Legacy Narrative Cleanup — REVORY Seller

Data: 2026-03-29  
Escopo: landing, setup, onboarding, dashboard, operational surfaces, templates e microcópias.

## Veredito
**Framing Seller ficou limpo e consistente.** As sobras de channel-first, recovery e review framing foram reduzidas para linguagem de booking-first, feedback e follow‑up, mantendo a honestidade do MVP.

## Substituições aplicadas

| Local | Texto antigo | Texto novo | Justificativa |
| --- | --- | --- | --- |
| Landing trust label | “Email-First · SMS-Ready” | “Email Default · SMS Optional” | Remove framing channel-first e evita prometer multicanal maduro. |
| Landing trust sub | “Fully usable on email from day one, with SMS available when needed” | “Fully usable on email from day one, with SMS optional when needed” | Reforça SMS como opcional, não core. |
| Landing section title | “Email-First Setup” | “Email-Default Setup” | Mantém email como default sem narrar canal como produto. |
| Landing feature copy | “Fully operational without WhatsApp.” | “Fully live without WhatsApp.” | Remove “operational” e reforça estado live. |
| Landing ROI copy | “noisy operational screens” | “noisy workflow screens” | Evita framing ops pesado. |
| Landing self-serve copy | “heavy ops” | “heavy overhead” | Evita linguagem ops. |
| Landing FAQ | “email-first and fully operational … SMS is available as an optional second channel” | “email-default and fully live … SMS is optional as a second channel” | Mantém promessa email-default e reduz linguagem de canal. |
| Setup booking path (labels) | “Email-first path / SMS-assisted path” | “Primary booking path (Email) / Assisted booking path (SMS)” | Centraliza booking path como decisão principal. |
| Dashboard metric | “Source paths active” | “Sources connected” | Remove jargão e fica mais premium. |
| Operational surface | “Ready next” | “Next step ready” | Linguagem mais humana e booking-first. |
| Operational surface copy | “execution engine” | “delivery engine” | Evita framing de execução pesada. |
| Operational categories | “Recovery opportunities” | “Follow-up opportunities” | Remove framing recovery. |
| Review category | “Reviews / Review request” | “Feedback / Feedback request” | Remove review framing, preservando o objetivo post‑visit. |
| Review blockers | “Missing reviews destination / Google Reviews destination” | “Missing feedback destination / Google feedback link” | Retira “reviews” do copy e mantém clareza funcional. |
| At‑risk reason | “operational attention” | “booking attention” | Mantém foco em booking, não ops. |

## Observações finais
- Nenhuma estrutura foi refeita; apenas linguagem user-facing foi alinhada.
- Mantida honestidade do MVP e o visual premium existente.

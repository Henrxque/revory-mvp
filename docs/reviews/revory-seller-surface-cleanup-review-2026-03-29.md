# REVORY Seller Surface Cleanup Review

## Executive Read

The product surface now reads much closer to a true `REVORY Seller` experience.

The cleanup focused on removing navigation and surface language that implied queue work, execution layers, or an operations stack. The visual shell stayed contained and premium; the shift happened in wording, emphasis, and how each surface presents itself.

## What Was Cleaned

- Sidebar navigation now reads:
  - `Revenue View`
  - `Lead Sources`
  - `Seller Setup`
- App header now uses `Seller live` and `Seller setup in progress`
- Imports hero now reads `Lead Sources` instead of `Sources & Mapping`
- Guided import flow now uses `final check`, `current import`, and `run import` instead of `execution` language

## Reinterpretations Applied

- `Operational Layer` -> `Booking Pulse`
- `Operational focus` -> `Conversion guidance`
- `Actionable next` -> `Ready next`
- `Next controlled step` -> `Next guided step`
- `Short focus list` -> `Booking priorities`
- `Message Foundation` -> `Booking Playbook`
- `Controlled preparation` -> `Guided preparation`

## Items Removed From The Center

- queue-like framing
- workbench-style naming
- execution-layer wording
- message-foundation framing
- operational-lane emphasis in primary navigation

## Alignment Check

The app now reads as:

- premium
- narrow
- booking-first
- guided
- self-service

It reads less like:

- recovery ops
- review ops
- outreach foundation
- pseudo-execution layer
- mini CRM

## Veredito

**Aprovado.**

The navigation and inherited surface language are now cleaner, narrower, and more consistent with the Seller thesis without inflating scope or introducing new heavy modules.
